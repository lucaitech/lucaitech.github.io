IAM & MFA Implementation

This project highlights my hands-on experience managing identity and access across two major platforms: Microsoft Entra (Azure AD) and Okta. My responsibilities included assigning roles, enforcing MFA, onboarding users, and ensuring access controls aligned with security policies. All examples below are fictionalized but accurately reflect the workflows I performed.

1. Role Assignment in Microsoft Entra

In Microsoft Entra, I regularly assigned and adjusted user roles based on job function, least-privilege principles, and access requests. This included both built-in roles and custom roles depending on the environment.

  • Assigning roles such as Global Reader, User Administrator, and Application Administrator
  • Reviewing access requests for accuracy and necessity
  • Ensuring users were placed in the correct security groups
  • Removing elevated roles when no longer required
  • Validating permissions after assignment
Example Role Assignment (Fictional):

User: jsmith@company.com
Requested Role: User Administrator
Verification: Confirmed with manager + ticket approval
Action: Role assigned, group membership updated
Follow-Up: Verified access by checking Entra audit logs

2. Okta MFA Setup & User Onboarding

I also handled MFA enrollment and troubleshooting in Okta. This included onboarding new users, enforcing MFA policies, and assisting employees with device or authentication issues.

  • Activating Okta accounts for new hires
  • Enforcing MFA enrollment (Okta Verify, SMS, or Email)
  • Resetting MFA factors when users changed devices
  • Unlocking accounts and clearing authentication attempts
  • Ensuring users completed MFA setup before accessing sensitive systems
Example MFA Setup (Fictional):

User: mgarcia@company.com
Issue: New phone → Okta Verify not transferring
Action: Reset Okta factors + re-enrolled user
Result: Successful MFA login validated through Okta logs

3. Access Control Improvements

By coordinating role assignments in Entra and MFA enforcement in Okta, I helped strengthen identity security and reduce unauthorized access risks. My work contributed to:

  • Cleaner role assignments aligned with least privilege
  • Consistent MFA coverage across the organization
  • Reduced authentication issues through proactive support
  • Improved audit readiness through accurate access records

4. Final Outcome

This project demonstrates my ability to manage identity lifecycle tasks, enforce MFA, and maintain secure access controls across multiple platforms. These workflows directly supported the organization’s security posture and compliance requirements.
