SIEM Alert Triage Playbook

This playbook outlines my structured approach to triaging security alerts in a SIEM environment. It reflects real workflows I’ve used in production environments, adapted with redacted and fictionalized examples for confidentiality.

1. Initial Alert Review

  • Identify alert type (malware, authentication anomaly, privilege escalation, network activity).
  • Check severity level assigned by the SIEM.
  • Review timestamp and affected host/user.
  • Determine whether the alert is part of a known pattern or campaign.
Example Alert:
“Suspicious PowerShell Execution Detected on HOST‑A12”
Severity: High
User: jdoe
Command: powershell.exe -enc …

2. Enrichment & Context Gathering

  • Check endpoint logs (CrowdStrike, Defender, etc.).
  • Review recent authentication activity for the user.
  • Look for related alerts within the same timeframe.
  • Check asset criticality (server, workstation, privileged user).
What I Look For:
  • Was the user logged in locally or remotely?
  • Is the PowerShell command obfuscated?
  • Has this host triggered alerts recently?
  • Any known vulnerabilities on the system?

3. Determine True vs. False Positive

I evaluate whether the alert represents real malicious activity or expected behavior.

  • Check if the user is part of IT or admin teams.
  • Verify if the command matches legitimate tools or scripts.
  • Look for signs of lateral movement or privilege escalation.

4. Response Actions

  • Isolate the host if malicious activity is confirmed.
  • Reset user credentials if compromise is suspected.
  • Block malicious IPs or domains.
  • Collect forensic artifacts (logs, processes, hashes).
  • Escalate to Tier 2/IR if needed.

5. Documentation & Closure

Every alert is documented with:

  • Summary of findings
  • Steps taken
  • Evidence collected
  • Final determination (TP/FP/Benign)
  • Recommendations for prevention
Sample Investigation Notes:
- Alert triggered by encoded PowerShell command.
- User is non‑technical; activity unexpected.
- Endpoint logs show connection to known malicious IP.
- Host isolated; password reset; case escalated to IR.
- Final Verdict: True Positive — Malware Execution
